System & Aggregates
GET /api/summary Host + VM count, avg CPU, warnings ▼
Returns a high-level aggregate across all registered hosts: total host count, total VM count, average CPU utilisation, and number of hosts currently in warning state (CPU or RAM > 85%).
Responses
200
{ "hosts": 4, "total_vms": 12, "avg_cpu": 34.2, "warnings": 1 }
▶ Try it out
GET /api/overview Detailed resource totals across all live hosts ▼
Returns aggregated resource metrics across all hosts that have live data: average CPU/RAM/disk, total RAM/disk used vs total, and counts of online hosts.
Responses
200
{ "hosts_online": 3, "total_hosts": 4, "avg_cpu": 41.5, "avg_ram": 62.1, "avg_disk": 55.3, "ram_total_gb": 256, "ram_used_gb": 158.9, "disk_total_gb": 8192, "disk_used_gb": 4510.2 }
▶ Try it out
GET /api/capacity Capacity planning — hardware vs VM allocation vs free ▼
Returns per-host capacity data: hardware totals (vCPUs, RAM, storage), VM allocations from running VMs only, and free resources. Used by the Capacity Planning page.
Responses
200
[{ "host_id": "h3a9f1b2c", "host_name": "dc-hyper-31", "host_ip": "192.168.101.32", "hw_missing": false, "cpu_model": "Intel Xeon E5-2690 v4", "cpu_sockets": 2, "cpu_pcores": 28, "cpu_vcpus": 56, "ram_total_gb": 256, "disk_total_gb": 4096, "disk_used_gb": 812.4, "vm_count": 8, "vm_running": 7, "vm_ram_alloc_gb": 119.5, "vm_vcpu_alloc": 32, "vm_disk_alloc_gb": 640.0, "free_ram_gb": 136.5, "free_vcpus": 24, "free_disk_gb": 2643.6, "vms": [{"name":"DC-JAVADEV-01", "status":"running", "ip":"192.168.101.45", "ram_gb":16, "vcpus":4, "disk_gb":80}] }]
▶ Try it out

Returns 24 hourly data points for CPU and RAM utilisation used to draw the area charts on the Overview page.

GET /api/alerts All active alerts sorted by severity ▼
Generates alerts from cached metrics and patch data. Alert types: Connection, CPU, RAM, Disk, Storage (volume), NIC Error, Patch, Security Patch. Critical alerts are returned first.
Responses
200
[{ "id": "a-h3a9f1b2c-cpu", "host": "dc-hyper-31", "type": "CPU", "severity": "critical", "msg": "CPU at 91%", "ts": "2025-01-15T08:32:00Z" }]

POST /api/test-connection Test SSH/WinRM before adding a host ▼
Tests connectivity to a host without saving it. Walks through TCP → SSH/WinRM → Metrics steps and returns pass/fail for each. Use this to validate credentials before adding a host.
Request Body
{ "name": "my-server", // required "ip": "192.168.1.10", // required "os_type": "linux", // "linux" | "windows" "username": "root", "password": "secret", "ssh_port": 22 }
Responses
200
{ "ip": "192.168.1.10", "steps": [ {"step":"TCP:22", "ok":true, "detail":"rc=0"}, {"step":"SSH", "ok":true, "detail":"authenticated"}, {"step":"Metrics","ok":true, "detail":"CPU:34% RAM:62%"} ] }
Hosts
GET /api/hosts List all registered hosts with metrics and VMs ▼
Returns all registered hosts with their cached metrics, VM list, and patch status. Passwords and SSH keys are masked.
▶ Try it out
GET /api/hosts/{hid} Get a single host by ID ▼
Path Parameters
NameTypeRequiredDescription
hidstringrequiredHost ID (e.g. h3a9f1b2c)
Responses
200Full host object with metrics, patch, and vms arrays
404Host not found
POST /api/hosts Add a new host and trigger initial data collection ▼
Registers a new host. Immediately collects metrics, patch status, and discovers VMs. Each step is independent — a failure in one does not block the others.
Request Body
{ "name": "dc-hyper-31", // required "ip": "192.168.101.32", // required "os_type": "linux", // "linux" | "windows", default "linux" "auth_type": "password", // "password" | "key" "username": "root", "password": "secret123", // for auth_type=password "ssh_key": null, // PEM string for auth_type=key "ssh_port": 22, "winrm_port": 5985 }
Responses
201
{"id":"h3a9f1b2c", "name":"dc-hyper-31", "connected":true, "vms_found":8, "message":"Connected ✔ — Ubuntu 22.04, 8 VMs found"}
400Database error saving host
PUT /api/hosts/{hid} Update host credentials or name ▼
Updates editable fields on a host. All fields are optional — only provided fields are updated.
Request Body (all optional)
{ "name": "new-name", "username": "admin", "password": "newpass", "ssh_key": "-----BEGIN RSA PRIVATE KEY-----...", "ssh_port": 2222 }
Responses
200{"id":"h3a9f1b2c","message":"Updated"}
404Host not found
DELETE /api/hosts/{hid} Remove a host and all its data ▼
Permanently deletes a host and cascades to remove all associated metrics, VMs, patch data, scan results, and logs.
Responses
200{"message":"Deleted"}
404Host not found
POST /api/hosts/{hid}/refresh Re-collect metrics, patch info, and VMs ▼
Triggers a full data refresh: metrics → patch → VMs. Each step is independent. If the host is unreachable, patch and VM steps are skipped to avoid long timeouts. Results are saved to DB.
Responses
200
{"status":"ok", "source":"live", "vms":8, "os":"Ubuntu 22.04 LTS", "message":"Refreshed — Ubuntu 22.04 LTS, 8 VMs"}
404Host not found
GET /api/hosts/{hid}/metrics Get cached metrics for a host ▼
Returns the most recently cached metrics snapshot for the host. Includes CPU, RAM, disk, storage, NICs, ports, OS info, and hardware details.
Responses
200Full metrics object including hardware{}, storage[], nics[], active_ports[]
404Host not found
GET /api/hosts/{hid}/patch Get cached patch status for a host ▼
Responses
200
{"status":"UPDATES AVAILABLE", "updates_available":14, "security_updates":3}
404Host not found
POST /api/hosts/{hid}/patch/refresh Re-collect patch data from the host ▼
SSHes into the host and runs apt/yum/dnf to get current package update status. Saves result to DB.
Responses
200Updated patch object
404Host not found
GET /api/hosts/{hid}/logs Get event logs for a host ▼
Returns the InfraCommand event log for a host — collection events, refresh actions, patch events, VM discovery. Up to 500 entries, newest first.
Responses
200Array of log entries with ts, level, msg, source fields
POST /api/hosts/{hid}/logs/refresh Pull live system logs from the host ▼
SSHes into the host and pulls the last 100 lines from journalctl / syslog / messages. Saves to DB and returns the entries.
Responses
200Array of log entries
500SSH connection failed
POST /api/hosts/{hid}/scan Run vulnerability scan on a host ▼
Runs a vulnerability scan against the host and saves the result. Returns CVE findings with severity, CVSS score, and affected package.
Responses
200Scan result with summary (critical/high/medium/low counts) and vulns[] array
404Host not found
GET /api/hosts/{hid}/scan Get stored scan result for a host ▼
Responses
200Stored scan result
404No scan — run a scan first
POST /api/hosts/{hid}/portscan Run a TCP port scan on a host ▼
Scans common TCP ports on the host's IP and returns open/closed status with service identification.
Responses
200
{"ip":"192.168.101.32", "ports":[{"port":22,"state":"open","service":"ssh"}], "scanned_at":"..."}
Virtual Machines
GET /api/hosts/{hid}/vms List VMs on a host ▼
Returns the cached list of VMs discovered on the host. Each VM includes status, IP, vCPU, RAM, disk, NICs, storage, and live metrics.
Responses
200
[{ "id": "vm-h3a9f1b2c-DC-JAVADEV-01", "name": "DC-JAVADEV-01", "status": "running", "ip": "192.168.101.45", "vcpu": 4, "ram_mb": 16384, "disk_gb": 80.0, "hypervisor": "KVM", "os": "Linux", "metrics": {"cpu":34.1,"ram":58.7}, "nics": [...], "storage": [...] }]
404Host not found
POST /api/hosts/{hid}/vms/refresh Re-discover VMs on a host ▼
Runs VM discovery via virsh (KVM) or Hyper-V PowerShell. Replaces the entire cached VM list for the host. Ghost guestfs-* domains are automatically filtered out.
Responses
200
{"count":8, "vms":[...]}
404Host not found
POST /api/hosts/{hid}/vms/{vid}/promote Promote a VM to a standalone monitored host ▼
Registers a discovered VM as its own standalone host entry so it can be monitored independently with full metrics, patch, and log collection.
Request Body
{ "username": "root", // required "password": "secret", // required "ssh_key": null, "port": 22 }
Responses
200
{"status":"ok", "host_id":"h7b2c3d4e", "message":"DC-JAVADEV-01 added as standalone host"}
400username and password required
404VM not found
PATCH /api/hosts/{hid}/vms/{vid}/ip Manually set a VM's IP address ▼
Allows manually overriding a VM's IP address. Useful for macvtap direct-mode VMs where automatic detection fails. The IP is flagged as manual_ip: true and preserved across refreshes.
Request Body
{"ip": "192.168.101.55"}
Responses
200
{"status":"ok", "ip":"192.168.101.55"}
400ip required
404VM not found
GET /api/hosts/{hid}/vms/{vid}/logs Get logs for a specific VM ▼
Returns VM-specific logs from InfraCommand's event store. Falls back to filtering host logs by VM name if no VM-specific entries exist.
Responses
200Array of log entries
POST /api/hosts/{hid}/vms/{vid}/scan Run vulnerability scan on a VM ▼

Responses

404Host or VM not found

Debug
GET /api/hosts/{hid}/debug/vm-ips Raw VM IP detection output for debugging ▼
SSHes into the host and runs all VM IP detection commands, returning their raw output. Useful for diagnosing why a VM's IP cannot be auto-detected. Returns: virsh list, domifaddr, domiflist, ARP table, ip neigh per VM.
Responses
200
{ "virsh_list": "Id Name State\n--- ---------- -------\n 1 DC-JAVADEV running", "vms": { "DC-JAVADEV-01": { "state": "running", "domifaddr": "...", "domiflist": "...", "macs": ["52:54:00:ab:cd:ef"], "arp_52:54:00:ab:cd:ef": "192.168.101.45 ether 52:54:00:ab:cd:ef" } } }
404Host not found
POST /api/debug/connect Full diagnostic — TCP, SSH, metrics with detailed errors ▼
Runs a complete connectivity and metrics diagnostic against any host. Returns a step-by-step result for TCP port check, SSH/WinRM auth, and metrics collection. Full exception details included — used by the Debug Console tab.
Request Body
{ "name": "debug", "ip": "192.168.101.32", "os_type": "linux", "username": "root", "password": "secret", "ssh_port": 22 }
Responses
200
{ "ip": "192.168.101.32", "steps": [ {"step":"TCP:22", "ok":true, "detail":"rc=0"}, {"step":"SSH", "ok":true, "detail":"authenticated"}, {"step":"Metrics", "ok":true, "detail":"CPU:34% RAM:62% Disk:55%"} ], "error": null }
